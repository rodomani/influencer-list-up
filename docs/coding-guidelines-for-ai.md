# コーディングAIのためのガイドライン

## 核となる原則

### 1. ユーザーの要件を正確に理解する

コードを書く前に、ユーザーが達成したいことを完全かつ正確に理解してください。

### 2. シンプルで簡潔なコードを書く

常に要件を満たす最もシンプルなソリューションを目指します。過剰な設計は避けます。

### 3. 付け足し付け足しを避ける

壊れたアプローチの上に修正を重ねないでください。うまくいかない場合は一歩引いて再考します。

### 4. 実装前に複数のアプローチを比較する

アプローチがうまくいかないとき、なぜかを理解せずにすぐに諦めないでください。代替アプローチを考え、比較してから実装します。

## まとめ

効率的な開発の鍵は: **理解** → **計画** → **実装** → **付け足し回避**

複数の小さな調整を繰り返していることに気づいたら、立ち止まって最初からアプローチを再考してください。
